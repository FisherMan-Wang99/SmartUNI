前端数据提交到后端可视化

更新说明: 此页面已更新为最新的数据结构，反映了活动经历和学校偏好的新设计。

1. 数据提交流程

前端表单
学生填写标化成绩、活动经历、学校偏好等信息
↓
数据收集与验证
onSubmit函数收集表单数据并进行验证
↓
数据结构组织
将数据组织成特定的JSON格式，包含学术成绩、活动经历和学校偏好
↓
后端接收处理
发送到http://192.168.3.99:8000/recommend

3. 数据填充逻辑

考试类型与成绩: 根据选择的考试类型(SAT/ACT)，将成绩填入对应字段
英语考试类型与成绩: 根据选择的英语考试类型(托福/雅思/多邻国)，将成绩填入对应字段
活动经历: 将各类经历的时长转换为数值类型，无填写则默认为0
学校偏好: 包括职业兴趣、位置、规模、学费滑块值、专业、师生关系、学校氛围和国际化程度

4. 代码示例

// 在 onSubmit 函数中组织数据
const requestData = {
  academics: {
    gpa,
    sat: testType === 'SAT' ? testScore : null,
    act: testType === 'ACT' ? testScore : null,
    toefl: englishTestType.includes('托福') ? englishScore : null,
    ielts: englishTestType.includes('雅思') ? englishScore : null,
    det: englishTestType.includes('多邻国') ? englishScore : null
  },
  activities: {
    research_months: parseInt(researchExperience) || 0,
    research_type: researchType,
    volunteer_months: parseInt(volunteerExperience) || 0,
    volunteer_type: volunteerType,
    internship_months: parseInt(internshipExperience) || 0,
    internship_type: internshipType,
    art_months: parseInt(artPractice) || 0,
    art_type: artPracticeType,
    full_time_work_months: parseInt(fullTimeWork) || 0,
    full_time_work_type: fullTimeWorkType,
    honors: honors
  },
  preferences: {
    career_interest: careerInterest,
    location: location,
    size: schoolSize,
    tuition: tuition,
    major: major,
    teacher_student_ratio: teacherStudentRatio,
    school_atmosphere: schoolAtmosphere,
    international_level: internationalLevel
  }
};

// 发送数据到后端
wx.request({
  url: "http://192.168.3.99:8000/recommend",
  method: "POST",
  header: { "Content-Type": "application/json" },
  data: requestData,
  success: (res) => {
    // 处理成功响应
  },
  fail: (err) => {
    // 处理失败情况
  }
});

重要提示:
  • 数据通过 HTTP POST 请求发送到后端服务器的 http://192.168.3.99:8000/recommend 地址
  • 内容类型为 application/json
  • 考试成绩根据用户选择的考试类型自动填入相应字段
  • 活动经历的时长被转换为整数类型，未填写则默认为0
  • 学费通过滑块选择，范围为10,000至200,000元
  • 所有必填项在提交前会进行验证，确保数据完整性
  • 数据发送成功后，会跳转到详情页展示推荐结果
